# Preset: 내러티브 RPG / 오픈월드 RPG

## 레퍼런스
위쳐3, 사이버펑크2077, AC 오디세이, 데이브더다이버, 발더스게이트3

## 장르 정의
세계가 살아있고, 선택이 의미를 가지며, 캐릭터가 기억에 남는 RPG.
"플레이어가 이 세계의 주민이 된 것처럼 느끼게 한다."

## 핵심 루프

```
[월드 탐험] → [NPC/사건 발견] → [조사/대화/전투] → [선택] → [결과가 세계에 남음]
     ↑______________________________________________________________|
     └─ [캐릭터 성장] → [더 복잡한 상황 접근 가능] → ...
```

### 감정 곡선 루프 (데이브더다이버 모델)
```
[긴장/집중 구간] → [해소/보상] → [휴식/탐색] → [새 긴장 유발]
예: 어려운 던전 → 클리어 → 마을 귀환 대화 → 새 사건 발견
```

## 필수 시스템

### 세계 반응성 (World Reactivity) — 핵심 차별화
```
위쳐3가 위대한 이유: "내 선택이 세계를 바꾼다"

구현 레벨별 반응성:
Level 1 (최소): 퀘스트 완료 후 NPC 상태 변화
Level 2 (보통): 지역 분위기/대사 변화, 신문/소문 반영
Level 3 (높음): 세력 관계도 변화, 퀘스트 간 연결 영향
Level 4 (위쳐): 수십 시간 후 선택이 엔딩에 영향

인디/소규모 권장: Level 2 충실히 구현 > Level 3~4 흉내
```

### 관계 시스템
```python
class RelationshipSystem:
    """
    동반자/NPC와의 관계를 수치로 관리
    but 수치를 유저에게 직접 보여주지 않는 것이 핵심
    """
    RELATIONSHIP_TIERS = {
        'hostile':    (-100, -50),
        'neutral':    (-49,  49),
        'friendly':   (50,   79),
        'trusted':    (80,   100),
    }
    
    TIER_EFFECTS = {
        'hostile':  ['unique_hostile_dialogue', 'attack_player'],
        'neutral':  ['default_dialogue'],
        'friendly': ['unique_dialogue', 'discount_trade', 'side_quest_unlock'],
        'trusted':  ['companion_route', 'secret_info', 'unique_ending'],
    }
    
    def get_tier(self, score: int) -> str:
        for tier, (low, high) in self.RELATIONSHIP_TIERS.items():
            if low <= score <= high:
                return tier
```

### 오픈월드 구조 (AC 오디세이 모델)
```
지역 설계 원칙:
1. 각 지역은 독립적인 미니 스토리 보유
2. 밀도 > 면적 (넓고 빈 공간보다 좁고 풍부한 공간)
3. 수직 탐험 요소: 지상/지하/수중/고지대
4. POI 간 거리: 도보 2~3분 이내에 발견 요소

지역당 필수 요소:
  - 메인 스토리 연결 퀘스트: 1~2개
  - 독립 사이드 퀘스트: 3~5개
  - 수집 요소: 5~10개 (과도하게 많지 않게)
  - 숨겨진 발견: 1~2개 (힌트 없이)
```

## 성장 시스템 (내러티브 RPG 특화)

### 스탯 vs 스킬 설계 원칙
```
내러티브 RPG의 성장 = "내 캐릭터가 어떤 사람이 되는가"

나쁜 성장 설계: 수치만 올라감 (HP+10, ATK+5)
좋은 성장 설계: 새로운 선택지가 열림

예:
  - 설득 스킬 마스터 → 전투 없이 보스 설득 가능
  - 의학 지식 → 시체에서 새 정보 발견
  - 잠입 마스터 → 아예 다른 루트로 퀘스트 진행
```

## 대화 & 선택 시스템

### 선택지 설계 원칙 (위쳐 스쿨)
```
위쳐3의 핵심 발견: "최선이 없는 선택이 가장 기억에 남는다"

나쁜 선택지: [선함] / [나쁨] / [중립]  ← 정답이 보임
좋은 선택지: [가족을 살림, 마을을 위험에 빠트림] / [마을을 지킴, 가족을 희생]

원칙:
1. 선택 시점에는 결과를 모름
2. 장기 결과가 단기 결과와 다를 수 있음
3. 도덕적 명확성을 피함
4. 감정적으로 어려운 결정이 기억에 남음
```

### 실시간 vs 타이머 대화
```
실시간 (제한 없음): 깊이 있는 대화, 정보 수집 퀘스트
타이머 대화:        긴장감 조성, 추격/위기 상황
침묵 선택지:        강렬한 감정 표현 (갓오브워 식)
```

## 데이브더다이버에서 배울 것

데이브더다이버가 특별한 이유 분석:

```
1. 장르 혼합의 대담함
   다이빙(액션) + 식당 경영(시뮬) + 낚시(미니게임) + 스토리
   → 각 장르 팬 모두 진입 가능

2. 진행 속도의 절묘한 배분
   "지루할 타이밍에 새 시스템 등장"
   → 30분마다 새로운 메커닉 또는 서사 전환

3. 비주얼 톤의 일관성
   2D 픽셀 + 밝고 따뜻한 색감 → 진지한 내용도 가볍게 소화

4. 부담 없는 실패
   다이빙 실패 = 약간의 패널티, 식당은 항상 진행
   → 스트레스 없이 계속 플레이하게 됨

설계 시사점:
  - 복수의 게임플레이 레이어를 교차 배치
  - 실패가 완전한 좌절이 아닌 "다시 하고 싶은" 설계
  - 장르 혼합은 각 요소의 핵심 재미를 지킬 때만 성공
```

## Edge Case (장르 특수)

1. **선택 마비 (Paradox of Choice)**: 선택지가 너무 많으면 아무것도 안 함 → 선택지 최대 4개
2. **레벨 게이팅**: 스토리 때문에 가고 싶은데 레벨 부족 → 레벨 권장이지 강제가 아님
3. **빌드 막힘**: 특정 빌드로 진행 불가 구간 → 모든 빌드의 대안 경로 보장
4. **대화 스킵 유혹**: 전투만 하고 싶은 유저 → 대화를 짧고 임팩트 있게 + 스킵 가능
5. **빈 세계 느낌**: 넓은데 아무것도 없다는 느낌 → 밀도 우선, 면적 나중
