# Preset: 시뮬레이션 (Simulation / Management)

## 장르 정의
도시 건설, 경영, 농장, 생존, 공장 자동화 등 시스템 관리가 핵심인 장르.
핵심: **시스템 마스터리** + **최적화 만족감** + **나만의 세계 구축**

---

## 핵심 루프

```
[자원 수집] → [건설/생산] → [자원 → 상위 자원 가공] → [확장] → [새 문제 발생] → [해결]
```

### 전형적 티어 구조
```
원자재(T1) → 가공재(T2) → 완성품(T3) → 수출/특수 아이템(T4)
```

## 필수 시스템 체크리스트

### 경제 시스템
- [ ] **자원 계층화**: 최소 3티어 원자재-가공-완성품
- [ ] **생산 속도 균형**: 병목(Bottleneck) 지점 의도적 설계
- [ ] **무역/교환 시스템**: 부족 자원 보충 수단
- [ ] **수요/공급 다이나믹**: NPC 구매자가 가격에 반응하는가?

### 건설 시스템
- [ ] **그리드 vs 자유 배치**: 장르 특성에 따라 결정
- [ ] **업그레이드 vs 교체**: 건물을 고도화 or 새 건물로 대체
- [ ] **범위 영향**: 건물이 주변에 버프/디버프 주는가?
- [ ] **도로/연결**: 자원 수송 경로 시스템

### 이벤트/위기
- [ ] **랜덤 이벤트**: 가뭄, 화재, 침략 등 주기적 위기
- [ ] **목표 시스템**: 단기/중기/장기 목표로 방향 제시
- [ ] **재난 방어**: 유저가 대비할 수 있는 시간 여유 제공

## 수치 템플릿

### 생산 사슬 밸런싱
```python
# 생산 시간 vs 가치 비율 설계
resources = {
    '나무':      {'time': 10, 'value': 1,   'tier': 1},
    '판자':      {'time': 20, 'value': 3,   'tier': 2, 'needs': {'나무': 2}},
    '가구':      {'time': 60, 'value': 12,  'tier': 3, 'needs': {'판자': 3}},
    '명품가구':  {'time': 180,'value': 50,  'tier': 4, 'needs': {'가구': 2, '금속': 1}},
}

def efficiency_check(resource_chain):
    """생산 사슬 효율 검증 - 상위 티어가 하위 티어보다 시간당 가치가 높아야 함"""
    for name, r in resource_chain.items():
        time_value = r['value'] / r['time']
        print(f"{name}: 시간당 가치 = {time_value:.3f}")
```

### 균형 인구/자원 비율 (도시 건설)
```
인구 100명당 필요 자원 생산:
- 식량: 2단위/분
- 주거: 인구 × 1 유닛
- 에너지: 인구 × 0.5 kW
- 물: 인구 × 0.8 단위/분

성장률: 자원 잉여율 × 인구 × 0.01 (월간)
이탈률: 자원 부족 시 인구 × 0.05 (월간)
```

## Edge Case (장르 특수)

1. **생산 병목 마비**: 하나의 자원 부족으로 전체 생산 정지 → 대체 공급루트 설계
2. **최적 빌드 고착**: 항상 동일한 배치가 최적 → 랜덤 지형 / 이벤트로 가변성 부여
3. **무한 자원 루프**: 교역 차익으로 자원 무한 증가 → 교역 쿨타임 / 거래량 상한
4. **스케일 붕괴**: 후반부 생산량이 너무 커서 UI가 버벅임 → 수치 압축(K/M 표기) + 성능 최적화
