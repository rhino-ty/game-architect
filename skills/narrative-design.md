# S-05: 내러티브 설계

## 목적
게임의 스토리·월드빌딩·캐릭터·대화 시스템을 설계한다.
갓오브워, 위쳐3, 사이버펑크2077처럼 서사가 게임플레이와 통합되는 구조를 목표로.

---

## 내러티브 설계 3층 구조

```
World (세계관)
  └─ Story (서사 / 플롯)
       └─ Character (캐릭터 아크)
            └─ Dialogue (대화 시스템)
```

---

## 1. 세계관 설계 (World Building)

### 월드 성경 (World Bible) 필수 항목

```markdown
## 세계관 성경

### 핵심 전제 (Core Premise)
- 이 세계의 근본 규칙은 무엇인가?
- 마법/기술/신/이념 등 세계를 움직이는 힘은?

### 역사 & 배경
- 현재 세계가 만들어진 결정적 사건 (3~5개)
- 유저가 직접 접하는 '현재' 상황

### 파벌 & 세력 관계
| 세력명 | 이념 | 주인공과의 관계 | 내부 갈등 |
|--------|------|---------------|---------|
| | | | |

### 세계의 규칙 (Rules of the World)
- 이 세계에서 절대 불가능한 것
- 이 세계에서만 가능한 것
```

### 세계관 일관성 체크
- [ ] 마법/기술 시스템의 한계가 명확히 정의되어 있는가?
- [ ] 경제·정치 구조가 서사를 지지하는가?
- [ ] 유저가 만나는 모든 NPC가 이 세계의 논리 안에 있는가?

## 2. 서사 구조 (Story Architecture)

### 3막 구조 (Three-Act + 게임 루프 통합)

```
Act 1 — Setup (플레이타임의 약 25%)
  ├─ Hook: 첫 5분 안에 핵심 갈등 제시
  ├─ World Introduction: 튜토리얼을 서사로 포장
  └─ Inciting Incident: 주인공의 동기 확립

Act 2 — Confrontation (약 50%)
  ├─ Rising Action: 주요 퀘스트 체인
  ├─ Midpoint Reversal: 기대를 뒤집는 반전
  └─ Dark Night of the Soul: 주인공의 최저점

Act 3 — Resolution (약 25%)
  ├─ Climax: 핵심 전투/결정
  ├─ Resolution: 세계의 변화 확인
  └─ Denouement: 여운 + 속편 복선
```

### 비선형 서사 설계 (오픈월드/선택형)

```
분기 유형:
A. 결과 분기 (Consequence Branching)
   선택 → 단기 결과 다름, 장기 결론은 수렴
   예: 위쳐3의 대부분 사이드 퀘스트

B. 경로 분기 (Path Branching)
   선택 → 다른 경험, 같은 목적지
   예: AC 오디세이 대화 선택

C. 엔딩 분기 (Ending Branching)
   핵심 선택들의 누적 → 다른 결말
   예: 사이버펑크2077 엔딩 (3~5종)

권장: A + C 조합 (개발 비용 vs 임팩트 균형)
```

### 서사 개발 비용 추정
```
분기 엔딩 1개 추가 = 메인 스토리 개발 비용의 15~25%
사이드 퀘스트 1개 = 메인 퀘스트 1개의 30~40% 비용
대사 녹음 추가 = 텍스트 비용의 5~10배
```

## 3. 캐릭터 설계 (Character Design)

### 캐릭터 성경 템플릿

```markdown
## [캐릭터명]

### 핵심 정체성
- 한 줄 요약:
- 외적 목표 (What they want):
- 내적 목표 (What they need):
- 결점 (Flaw):
- 두려움 (Fear):

### 캐릭터 아크
- 시작 상태:
- 전환점 (Act 2 중반):
- 최종 상태:

### 관계 지도
- 주인공과의 관계:
- 빌런과의 관계:
- 내부 갈등:

### 말투 & 목소리
- 어휘 수준:
- 말버릇/독특한 표현:
- 감정 표현 방식:
```

### 강력한 빌런 설계 원칙
```
좋은 빌런 = "이해는 가지만 동의할 수 없는" 인물

체크:
- 빌런의 논리가 특정 시각에서는 맞는가?
- 빌런이 자신을 악당이라 생각하지 않는가?
- 빌런의 목표와 주인공의 목표가 표면적으로 유사한가?

레퍼런스: 위쳐3 겐트의 왕 / 갓오브워 발두르 / 사이버펑크 슈퍼마리오
```

## 4. 대화 시스템 (Dialogue System)

### 대화 설계 원칙

```
1. 보여주기 vs 말하기 (Show, Don't Tell)
   BAD:  "나는 아버지를 잃고 복수를 맹세했습니다."
   GOOD: [아버지의 무덤 앞에서 아무 말 없이 무기를 집어 드는 주인공]

2. 대화의 역할 분리
   - Exposition 대화: 세계 정보 전달 (10% 미만)
   - Character 대화: 관계·성격 표현 (40%)
   - Plot 대화: 사건 전진 (30%)
   - Choice 대화: 유저 에이전시 부여 (20%)

3. 대화 속도
   - 텍스트 평균 읽기 속도: 분당 200~250단어
   - 음성 평균: 분당 130~150단어
   - NPC 대화 1회 최대: 3~4문장 권장
```

### 대화 선택지 설계 패턴

```
패턴 A: 감정 선택 (위쳐 스타일)
  [공감] / [냉정] / [분노] / [유머]
  → 결과: 관계 수치 변화, 단기 대화 분기

패턴 B: 정보 수집 (어쌔신크리드 스타일)
  [질문1] / [질문2] / [질문3] / [떠나기]
  → 결과: 월드 지식 획득, 퀘스트 단서

패턴 C: 결정 (사이버펑크 스타일)
  [도덕적] / [중립] / [비도덕적]
  → 결과: 장기 서사 분기
```

## 5. 퀘스트 설계

### 퀘스트 품질 기준 (위쳐3 기준)

```
AAA 사이드 퀘스트 필수 요소:
1. 독립적인 미니 스토리 (메인과 연결 안 돼도 완결성 있음)
2. 도덕적 딜레마 (명확한 정답이 없음)
3. 결과가 세계에 남음 (퀘스트 완료 후 NPC/장소 변화)
4. 유머 or 감동의 감정 피크 1개

인디/AA 현실적 기준:
1. 명확한 목표 + 의외의 전개
2. NPC 1~2명의 기억에 남는 순간
3. 보상이 순수 수치 이상의 의미
```

### 퀘스트 유형별 개발 비용 가이드
```
Kill/Fetch Quest:   개발 1 (기준)
Escort Quest:       개발 1.5 (AI 조율)
Investigation:      개발 2 (환경 배치 + 단서 로직)
Multi-Ending Quest: 개발 3~4 (분기 + 스크립팅)
```
