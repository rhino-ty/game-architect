# S-01: 시스템 메커니즘 설계

## 목적
새로운 전투·성장·경제·사회 시스템의 구조를 제안하고, 숨겨진 Edge Case를 선제적으로 탐지한다.

---

## MDA 분석 프레임

설계 전 반드시 아래 3층을 정의한다:

```
Mechanics  →  Dynamics  →  Aesthetics
(규칙/수치)    (상호작용)      (경험/감정)
```

| 레이어 | 정의 | 예시 |
|--------|------|------|
| Mechanics | 시스템의 규칙과 수치 | 쿨타임 5초, 데미지 공식 |
| Dynamics | 규칙들이 만드는 패턴 | 탱커-딜러-힐러 시너지 |
| Aesthetics | 유저가 느끼는 감정 | 짜릿한 역전, 전략적 긴장감 |

---

## 시스템 설계 체크리스트

### 전투 시스템
- [ ] 기본 공격 공식 정의: `DMG = ATK × (1 - DEF_RATE)`
- [ ] TTK(Time To Kill) 산출: `TTK = HP / DPS`
- [ ] CC기(군중제어) 유무 및 카운터 설계
- [ ] 스킬 리소스 관리 방식 (MP, 쿨타임, 스택)
- [ ] 치명타/회피 확률 캡 설정 (권장: 치명타 ≤ 75%)

### 성장 시스템
- [ ] 성장 곡선 형태: 선형 / 지수 / S커브 / 로그
- [ ] 레벨 구간별 체감 강도 차이 설계
- [ ] 소프트 캡 / 하드 캡 위치 결정
- [ ] P2W vs F2P 성장 격차 상한선

### 경제 시스템 (Faucet & Sink)
- [ ] 재화 유입원(Faucet) 나열: 퀘스트, 일일보상, PvP 등
- [ ] 재화 소비처(Sink) 나열: 강화, 상점, 거래세 등
- [ ] 인플레이션 방어 장치: 공급 상한, 시즌 리셋, 거래세
- [ ] 재화 종류별 희귀도 계층 (일반화폐 / 프리미엄 / 바운드)

---

## Edge Case 탐지 패턴

설계 완료 후 반드시 아래 질문을 던진다:

1. **무한 루프 공격**: "이 스킬을 A→B→A로 반복하면?" → 쿨타임/소모 검증
2. **스케일링 붕괴**: "스탯이 최대치일 때 공식이 음수/0이 되지 않는가?"
3. **경제 익스플로잇**: "한 유저가 재화를 무한 생성할 수 있는 경로가 있는가?"
4. **PvP 균형**: "같은 전략만 쓰는 메타가 고착화될 가능성이 있는가?"
5. **신규 유저 장벽**: "Day 1 유저가 기존 유저와 맞닥뜨리는 시점은?"
