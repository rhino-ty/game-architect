# S-08: 게임 느낌 & 출시 준비 (Game Juice / Accessibility / Playtesting / Steam Copy)

## 목적
시스템과 수치가 아무리 좋아도, 이 4가지가 부족하면 리뷰에서 죽는다.
- **Game Juice**: "조작감이 살아있다" vs "조작감이 죽어있다"의 차이
- **접근성**: Steam/콘솔 제출 기준 강화 중, 유저 이탈 방지
- **플레이테스팅**: 개발자 눈엔 안 보이는 문제를 발견하는 체계
- **Steam 카피라이팅**: 게임을 팔게 해주는 텍스트

---

## Part 1: Game Juice (게임 느낌)

### 정의
Game Juice = 게임플레이 액션에 연결된 시각/청각/물리적 피드백 레이어.
수치가 같아도 Juice의 차이로 "쾌감"이 10배 달라짐.

### 피드백 레이어 체크리스트

#### 타격감 (Hit Feel)
```
□ 히트스톱 (Hitstop)
  일반 공격:   0.04~0.07초 프리즈
  강 공격:     0.08~0.14초 프리즈
  보스 처치:   0.2~0.3초 프리즈
  너무 길면:   답답함 | 너무 짧으면: 타격감 없음

□ 화면 흔들림 (Camera Shake)
  타격 시:     강도 2~5, 지속 0.1~0.2초
  폭발:        강도 5~10, 지속 0.2~0.4초
  ※ 접근성 옵션 필수 (전정 장애 유저)

□ 히트 이펙트
  파티클:      피가 튀거나 스파크, 방향성 있게 (타격 방향 반대로)
  적 플래시:   피격 시 흰색 플래시 0.05~0.1초
  데미지 숫자: 크기/색상으로 치명타 구분

□ 사운드 레이어링
  공격 SFX:    공격 시작음 + 타격음 분리
  치명타:      별도 SFX (높은 피치 or 중후한 베이스)
  사망:        적 사망 전용 SFX
```

#### 이동감 (Movement Feel)
```
□ Coyote Time (코요테 타임)
  낭떠러지 끝에서 0.08~0.15초 동안 점프 가능
  없으면: "분명히 타이밍 맞았는데 왜 안 됨?" 불만

□ Jump Buffer (점프 버퍼)
  착지 직전 0.1~0.2초 내 점프 입력 → 착지 즉시 점프
  없으면: 박자감 있는 점프가 어려움

□ 착지 이펙트
  먼지 파티클 + 착지음 + 카메라 미세 흔들림
  높이 비례해서 강도 조절

□ 대쉬 느낌
  스트레치 (대쉬 방향으로 캐릭터 늘어남)
  잔상 이펙트 (0.1~0.2초 잔상 3~5개)
  SFX + 가속감 있는 파티클
```

#### UI/UX 피드백
```
□ 버튼 클릭 반응
  시각: 누름 애니메이션 or 색상 변화
  청각: 경쾌한 클릭 SFX
  촉각: 컨트롤러 진동 (경량)

□ 보상 표시
  아이템 획득: 팡파르 SFX + 화면 플래시
  레벨업: 전신 파티클 + 음악 스팅
  도전과제: 고유 SFX + 배너 애니메이션

□ HP 변화
  감소: 빨간 비네팅 + 화면 흔들림 (강도 비례)
  위험 (20% 이하): 심박수 SFX + HUD 강조 색상
  회복: 녹색 파티클 + 상승음
```

### Game Juice 구현 우선순위

```
1순위 (반드시): 히트스톱, 히트 이펙트, 이동 SFX
2순위 (강력 권장): 카메라 흔들림, Coyote Time, Jump Buffer
3순위 (폴리시): 잔상, 착지 파티클, UI 애니메이션
```

## Part 2: 접근성 설계 (Accessibility)

### 왜 중요한가
- Steam 리뷰에서 "접근성 옵션 잘 돼 있다" = 긍정 리뷰 유발
- PlayStation 5: 접근성 가이드라인 공식화 (콘솔 포팅 시 필수 검토)
- 전체 게이머의 약 20%가 어떤 형태의 장애를 가짐

### 필수 접근성 옵션

#### 시각
```
□ 자막/텍스트 크기 조절 (최소 3단계)
□ 색약 모드 (Red-Green, Blue-Yellow, 흑백)
□ 배경/텍스트 대비 조절
□ 화면 흔들림 on/off (전정 장애)
□ 번쩍임/플래시 on/off (광과민성 발작)
□ HUD 투명도 조절
```

#### 조작
```
□ 버튼 리매핑 (키보드 + 게임패드 모두)
□ 홀드 vs 토글 전환 (버튼 장시간 누름 불필요하게)
□ 입력 유예 시간 조절 (반응 속도 보조)
□ 연속 입력 vs 단일 입력 전환
□ 마우스 감도 세분화
```

#### 난이도 접근성
```
□ 난이도 조절 (스토리/이지/노멀/하드/커스텀)
□ 사망 패널티 경감 옵션
□ 보스 전 자동 세이브
□ 무적 모드 (스토리 집중 유저용)
  ※ "이건 치트"가 아니라 접근성 — Celeste의 방식이 정석
```

#### 청각
```
□ 자막 on/off (음성 있는 게임)
□ 음성/SFX/BGM 독립 볼륨
□ 시각적 사운드 표시 (청각 장애 보조)
  예) 총소리 발생 방향을 화면에 표시
```

### Celeste의 접근성 설계가 정석인 이유
```
Celeste Assist Mode:
  - 무적 모드 / 공기 대시 무한 / 게임 속도 조절
  - 게임 시작 시 당당하게 메뉴에 표시
  - "이것도 정당한 플레이 방식"이라는 메시지

결과:
  - 접근성 배려로 부정 리뷰 최소화
  - 더 넓은 유저층 확보
  - 긍정 리뷰에 접근성 언급 다수
```

## Part 3: 플레이테스팅 프레임워크

### 왜 필요한가
개발자는 자기 게임을 알기 때문에 보이지 않는 것들:
- 직관적이지 않은 UI
- 어디로 가야 하는지 모르는 구간
- 밸런스 이상 (개발자는 익숙해서 쉽게 느낌)
- 첫 30분 이탈 지점

### 플레이테스트 3단계

#### 1단계: 내부 테스트 (알파 전)
```
목적: 크리티컬 버그 + 기본 플로우 검증
테스터: 개발자 본인 + 친한 지인 1~2명

측정:
  □ 튜토리얼 없이 핵심 조작 이해 가능한가?
  □ 첫 10분 안에 이탈하는 지점이 있는가?
  □ 크래시/무한루프 발생 지점
  □ "이게 뭐야?" 발생 지점 (UI 불명확)
```

#### 2단계: 외부 테스트 (알파~베타)
```
목적: 실제 유저 반응 수집
테스터: 타겟 페르소나와 유사한 낯선 사람 5~10명
        (친한 사람은 좋다고만 함)

방법:
  □ Think Aloud (생각을 소리내어 말하게 함)
  □ 개발자는 절대 개입하지 않음
  □ 화면 녹화 + 테스터 표정 녹화

측정:
  □ 포기하는 지점
  □ 기대와 다른 반응을 보이는 순간
  □ 가장 재밌어하는 순간
  □ 혼란스러워하는 UI/조작
  □ 플레이 시간 (1시간 이상 → 기본 합격)
```

#### 3단계: Steam 넥스트 페스트 데모 (출시 전)
```
목적: 대규모 공개 테스트 + 위시리스트 확보
데이터 수집:
  □ 데모 완료율 (얼마나 많이 끝까지 플레이?)
  □ 위시리스트 전환율
  □ 플레이 시간 분포
  □ Steam 토론에 올라오는 피드백

핵심 지표:
  데모 완료율 50%+ → 합격
  위시리스트 전환율 5%+ → 좋음
  데모 평균 플레이 30분+ → 핵심 루프 작동
```

### 플레이테스트 설문 템플릿

```
1. 첫 인상은? (5점)
2. 조작이 직관적이었나요? (5점)
3. 어디서 가장 막혔나요? (서술)
4. 가장 재밌었던 순간은? (서술)
5. 다시 플레이하고 싶나요? (5점)
6. 얼마를 내고 사겠나요? ($______)
7. 친구에게 추천하겠나요? (NPS 0~10)

NPS 해석:
  9~10: 프로모터 (강한 입소문)
  7~8:  패시브 (그냥 즐김)
  0~6:  디트랙터 (부정 리뷰 가능)
```

## Part 4: Steam 페이지 카피라이팅

### Short Description (한 줄 설명, 300자 이내)
Steam 게임 목록에서 보이는 첫 텍스트. **가장 중요한 카피**.

```
공식: [장르 키워드] + [핵심 훅] + [차별화 포인트]

나쁜 예:
  "흥미진진한 액션 RPG 게임. 다양한 캐릭터와 함께 모험을 떠나세요!"
  → 구체성 없음, 수백 개 게임과 동일

좋은 예:
  "죽어도 기억이 남는다. 매 런마다 세계가 조금씩 달라지는 내러티브 Roguelite."
  → 훅 명확, 차별화 즉시 전달

작성 원칙:
  □ 첫 문장에 훅 (질문, 역설, 강렬한 이미지)
  □ 장르를 직접 명시 (알고리즘 노출)
  □ "다양한", "흥미진진한" 형용사 지양
  □ 경쟁작보다 뭐가 다른지 1~2가지만
```

### Long Description (상세 설명)
```
구조:
  단락 1 (2~3줄): 핵심 컨셉 + 감정적 훅
  단락 2 (3~4줄): 핵심 게임플레이 특징 (bullet point)
  단락 3 (2~3줄): 차별화 포인트
  단락 4 (1~2줄): 타겟 유저에게 말 걸기

Bullet Point 작성법:
  나쁨: "다양한 무기와 아이템"
  좋음: "10종의 무기, 각각 고유한 전투 스타일 — 같은 보스도 매번 다르게 클리어"

주의:
  □ 마케팅 문구 아닌 사실 기반으로
  □ 1인칭 유저 시각으로 ("당신은 ...")
  □ 길면 읽지 않음 — 핵심 3~5가지만
```

### 스크린샷 전략 (10장 기준)
```
1번 (가장 중요): 게임의 핵심 순간 — "이 게임의 정수"
2~3번:          전투/액션 — 가장 화려한 순간
4~5번:          탐험/세계관 — 세계가 넓다는 느낌
6~7번:          UI/성장 시스템 — 깊이감 표현
8~9번:          스토리/감정 — 내러티브 있음 표현
10번:           숨겨진 요소 암시 — "더 있다"는 느낌

원칙:
  □ HUD는 최소화하거나 끔
  □ 고해상도 (1920x1080 이상)
  □ 글자 없는 스크린샷 (텍스트는 설명글로)
  □ 게임 내 가장 아름다운 순간 선별
```

### 트레일러 구조 (60초)
```
0~5초:   강렬한 훅 (말 or 행동, 음악 시작)
5~15초:  세계관/분위기 — "이런 세계야"
15~35초: 핵심 게임플레이 — 빠른 편집, 타격감 SFX
35~50초: 감정적 순간 — 스토리 힌트 or 보스 처치
50~55초: 차별화 포인트 텍스트 (3개 이내)
55~60초: 로고 + 출시일 or "Coming Soon"

음악 선택:
  인디 액션: 긴장감 있는 오케스트라 or 일렉트로닉
  내러티브: 감성적인 피아노 or 현악
  Roguelite: 중독성 있는 반복 비트
```
